Minecadia Bots

A Next.js dashboard for managing the Minecadia Discord bot fleet—fleet controls, ticket management, games administration, member analytics, and moderation tools, backed by MySQL and a Python FastAPI control layer.

Next.js / TypeScript MySQL FastAPI Discord OAuth Production
View on GitHub README

Overview

Minecadia runs multiple Discord bots (games, tickets, management, and more). This repo is the operator console: staff authenticate with Discord OAuth, inspect live data from MySQL, and drive bot processes through a local control API instead of SSH-ing into tmux sessions by hand.

What I built

  • Fleet control — restart bots, edit configs, and attach to tmux consoles via the Python control-api service.
  • Tickets & games admin — dashboard surfaces for ticket workflows and games bot settings, wired to per-bot HTTP APIs.
  • Analytics & engagement — member message rollups with deduplicated history scans and live ingestion from Management.
  • Moderation tooling — guild-aware views backed by Discord API tokens and read/write MySQL roles.
  • Deploy pipeline — npm run build / npm run deploy plus scripts for analytics backfill and schema migrations.

Stack

src/ is the Next.js app (pages, API routes, components). control-api/ is FastAPI for process control. Bot runtime code lives under MINECADIA_BOTS_ROOT outside this repo.

Data layer

Separate read and write DB credentials; analytics backfill respects load thresholds and message-ID dedup via analytics_member_messages_seen.

Technical notes

Requires Node.js 20.9+, MySQL, and a Discord application (OAuth + bot token). Environment covers guild ID, control API secrets, and per-bot API endpoints for Games and Tickets.

Design choice: keep dashboard UI in Next.js while process orchestration stays in FastAPI—so the web app never shells into production hosts directly, but operators still get one pane of glass for the whole fleet.
