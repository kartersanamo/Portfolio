Change Tables

A lightweight Python utility that ingests PLC export text files and applies configurable transformation rules from rules.json—streamlining data conversion workflows for the ECO automation team.

Python 3 Tkinter GUI PLC / SCADA Open source
View on GitHub README

Overview

PLC exports often need repeatable, rule-driven edits before they land in Proficy, Cimplicity, or other downstream tooling. Change Tables turns that work into a small desktop app: pick an input .txt, choose an output path, and apply a rules file—no code changes when the conversion recipe evolves.

What I built

  • Tkinter workflow — select input file, output destination, and optional custom rules.json, then click Apply Changes.
  • Line rules — when a line contains a find string, replace the entire line with replace (evaluated top to bottom).
  • Global rules — find-and-replace across the full file after line rules run.
  • word_30_fallback — optional pass that converts leftover *_WORD_30 tokens (e.g. FOO_WORD_30 → FOO_WORD 1).
  • Data-driven config — operators add, remove, or edit entries in JSON; the engine stays the same.

Architecture

Processing is split across rules.py (rule loading), engine.py (transformation pipeline), and change_tables.py with main.py wiring the GUI—keeping conversion logic separate from the interface.

Rules format

rules.json holds lines, global, and word_30_fallback. Teams can version rule sets per project or site without touching Python.

Run it

Requires Python 3 and tkinter (bundled with most installs). From the repo root:

python3 main.py — choose input .txt, output path, and apply. Rules load from rules.json by default or from a file you pick in the GUI.
