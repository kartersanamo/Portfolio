Rift

Rift is a production-ready global warp plugin for Spigot servers, built around fast command flow, GUI-first management, configurable teleport UX, and resilient YAML persistence for real-world server operations.

Spigot / Bukkit API Java 21 Maven Production ready
View on GitHub Release v1.0.0

Why this project

Many warp plugins stop at basic commands, but production servers need stronger guardrails: configurable teleport timing, informative validation, full GUI controls, and safe behavior when worlds or locations go missing. Rift was built to deliver that full operational layer.

What I built

  • Complete warp lifecycle - create, update, browse, inspect, and delete warps through both commands and GUI actions.
  • Full command suite - /setwarp, /warp, /deletewarp, /warps, /warpinfo, and /rift reload with tab completion.
  • Interactive GUI workflow - browse warps, then edit name, material, description, category, and location through guided in-game flows.
  • Message-driven configuration - 130+ configurable keys in messages.yml with placeholders and fallback defaults.
  • Persistence and safety - YAML-backed warp metadata (creator, timestamp, usage count) with graceful loading and backup support.

Architecture

Rift separates framework utilities (chat, config, commands, GUI base classes) from plugin-specific modules (warp, command handlers, GUI screens). Shared validation and chat-input abstractions reduce duplicate logic across command and GUI paths.

Operator experience

Server owners can tune teleport delay effects, naming constraints, description limits, and all user-facing language without recompiling. The plugin also handles invalid world/location data gracefully instead of breaking startup.

Technical notes

The project uses Java 21 + Maven with Spigot dependencies and a soft dependency on Multiverse-Core. GUI management includes pagination and admin utilities (stats, reload, timestamped backups), and teleports can be canceled on movement for safer gameplay flow.

Result: Rift reached a clean production-ready state with tested command wiring, GUI actions, configurable messaging, and stable package output for direct server deployment.
