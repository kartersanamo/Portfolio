Minecadia Games

A community hub bot that turns Discord into an arcade: persistent XP and levels, a real-time leaderboard, and a growing library of games—built to stay fun under real concurrent load.

discord.py Async Python PostgreSQL-ready patterns Live product
View on GitHub Join Discord

Overview

Minecadia Games anchors engagement for the Minecadia Discord: members earn XP for participation, climb a visible leaderboard, and drop into lightweight games without leaving the server. The goal was retention mechanics that feel fair, transparent, and easy to moderate.

What I built

  • Progression core — XP awards, level curves, and per-member state with clear rules for farming prevention.
  • Leaderboard — live-updating presentation (embeds / components) so rankings stay readable as the member list grows.
  • 10+ game modes — modular command and session flow so new games plug in without rewriting the progression layer.
  • Operational hooks — staff-facing behaviors and logging-oriented structure for a busy community server.

Engineering

discord.py’s async model drives the bot; work is split into cohesive cogs/services so game logic doesn’t entangle with transport code (interactions, messages, rate limits).

UX

Copy, error paths, and command discoverability were tuned for players who won’t read a manual—short feedback loops and consistent embed formatting.

Outcomes

The bot supports a long-running community workload: concurrent sessions, predictable command latency, and room to extend seasonal events or new mechanics without breaking the leveling contract players rely on.

Takeaway: treating XP rules as a small domain model (inputs, caps, events) early made balancing and debugging much cheaper than scattering arithmetic across commands.
