Raid Riot

Spigot 1.8.8 faction raiding event plugin. Two teams compete in a dedicated arena—the first to breach the enemy base wins, or the team with greater wall penetration when time expires.

Java Spigot 1.8.8 SaberFactions WorldEdit (optional)
View on GitHub

Why this project

Faction servers need structured PvP events that feel fair, repeatable, and operator-friendly. Raid Riot automates the full match lifecycle—from queue and team assignment through arena setup, live breach tracking, and terrain restore—so staff can run competitive raiding events without manual world cleanup or ad-hoc rule enforcement.

Event lifecycle

Matches flow through a defined state machine: IDLE → QUEUE_OPEN → QUEUE_LOCKED → [VOTING] → PREPARING → COUNTDOWN → ACTIVE → ENDING → RESTORING. Voting is skipped when fixed match settings are enabled in config. Admins can stop a session at any point.

What I built

  • Queue system — Random mode assigns teams when the queue locks at capacity or countdown expiry; Faction mode fills teams from the first two factions to reach players-per-team members.
  • Voting GUI — Participants vote for base difficulty (Easy, Medium, Hard schematic, or live faction base copy) and kit (predefined or own inventory at match start).
  • Arena preparation — Schematic or faction-base paste, system faction claims, world border resize, inventory/location snapshots, and kit application at countdown end.
  • Match rules — Breach and depth-based win conditions, TNT attribution, virtual death with timed respawn, building restrictions, and in-match /f claim routed to event team factions.
  • Spectator mode — Non-participants spectate via GUI with player-head teleport; full state snapshot on enter/leave.
  • Async world restore — Block deltas tracked as non-air chunk snapshots and restored after matches with configurable blocks/chunks per tick.

Win conditions

Breach — attacker penetrates past the wall plane (depth ≥ 1). Depth — higher recorded wall penetration when the timer expires. Draw — equal depth when enabled. Admins can also force-stop a match.

Player interface

/raidriot (/rr) opens queue, voting, match status, and spectator GUIs with live refresh. All messages and GUI strings live in config.yml via ConfigManager.

Technical notes

SaberFactions is required at runtime via reflection; WorldEdit is optional for schematic paste. System factions (configurable tags such as Yellow/Red) claim base territory during events. Breach detection feeds from block breaks, explosions, and dispenser/spawn TNT attribution against each team's wall region. Combat toggles SaberFactions friendly-fire metadata so cross-team PvP works within the same event faction.

Result: A full-featured raiding event framework with operator controls, player-facing GUIs, configurable gameplay, and safe post-match terrain restore for production Factions servers.
