Minecadia Utilities

A Python Discord bot powering community utilities for the Minecadia network—polls, tags, suggestions, screenshare logging, message tracking, player counts, and Telegram integration.

discord.py Python MySQL Live product
View on GitHub Join Discord

Overview

While Games and Tickets handle play and support, Utilities is the day-to-day glue for Minecadia: community input (polls, suggestions), staff workflows (tags, embed builder), and cross-platform signals (player counts, Telegram). It is built as a cog-based discord.py bot with clear separation between UI, services, and data access.

What I built

  • Community tools — polls, tag management, suggestions, and an embed builder for consistent announcements.
  • Staff & moderation helpers — screenshare logging, message counter, sync tools, and utility commands for operators.
  • Live server metrics — player count integration so Discord reflects Minecraft network status.
  • Telegram bridge — integration hooks so alerts and workflows can reach channels outside Discord.
  • Configurable deployment — .env for secrets and assets/config.json for channels, roles, and Trello boards.

Architecture

Organized into cogs/, core/, services/, repositories/, and ui/—commands stay thin while business logic and persistence live in dedicated layers.

Operations

Standard pip install -r requirements.txt, .env from .env.example, and python main.py (or run.sh) for production-style launches on the fleet host.

Part of the fleet

Utilities runs alongside Minecadia Games, Tickets, and Management—managed from the Bots dashboard when operators need restarts, configs, or analytics context.

Takeaway: a utilities bot succeeds when every feature is independently togglable via config—channels, roles, and webhooks should not require code edits when staff reorganize the server.
