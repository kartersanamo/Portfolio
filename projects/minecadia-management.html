Minecadia Management

A Python Discord bot for internal management and moderation on the Minecadia server—timeouts, mention alerts, moderation logs, media checks, analytics, and staff tooling.

discord.py Python MySQL Live product
View on GitHub Join Discord

Overview

Management is the staff-facing bot in the Minecadia fleet: it enforces moderation workflows, keeps audit trails, and feeds analytics that power the Bots dashboard. Message events from this bot integrate with member engagement rollups and backfill pipelines.

What I built

  • Moderation — timeouts, mention alerts, and structured moderation logs for accountability.
  • Media compliance — YouTube channel checker for submissions using the YouTube Data API.
  • Analytics — server analytics and member message tracking persisted to MySQL.
  • Staff commands — analysis and management commands scoped to roles and channels from config.
  • Layered codebase — same cog / service / repository layout as Utilities for maintainability across the fleet.

Configuration

.env holds the Discord token, database credentials, and YOUTUBE_API_KEY. assets/config.json defines channels, roles, and embed settings without redeploying code.

Deployment

pip install -r requirements.txt, configure .env from .env.example, then python main.py or run.sh on the production host alongside the rest of the Minecadia bots.

Technical notes

Live message ingestion writes through the same deduplication paths used by dashboard analytics—so historical backfills and real-time counts stay consistent after deploys.

Takeaway: moderation bots need observable actions—every timeout, alert, and log entry should be traceable to config, staff role, and target context for disputes weeks later.
