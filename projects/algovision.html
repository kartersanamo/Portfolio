AlgoVision

Turn a 3D block world into a live canvas for algorithms: define a region in-game, run a visualization, and watch data structures behave—with particles, sound, and timing you can feel.

Concept

Classic algorithm teaching often stays abstract. AlgoVision bridges intuition and implementation: students (or curious players) select space in the world, then algorithms manipulate blocks or effects in sync with each step of the computation.

What I built

  • Spatial selection — workflow to define and validate regions suitable for grid-based visuals.
  • Algorithm library — sorting, searching, graph traversals, pathfinding, and other patterns exposed through a consistent runner interface.
  • Sensory feedback — deliberate use of particles and audio so state changes are visible even from a distance.
  • Scheduling — tick-safe animation so the server stays responsive while demonstrations play out.

Rendering model

Step-wise updates map abstract algorithm events to block/particle operations, keeping the “physics” of the visualization decoupled from algorithm correctness.

Extensibility

New algorithms plug into the same pipeline—focus stays on the core steps and how they should be perceived in-world.

Why it matters

AlgoVision is as much a software architecture exercise as a gameplay gimmick: bounded regions, cancellable runs, and predictable cleanup are mandatory when users can wander into half-finished animations.

Takeaway: the hardest part wasn’t the algorithms—it was making every frame of the animation reversible enough for multiplayer safety and server tick budgets.
