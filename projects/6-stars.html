Six Stars Hotel

Six Stars Hotel is a single-property desktop client for managing reservations, guest stays, billing, and account flows. It uses a Swing UI, SQLite persistence, and role-based navigation for guests, clerks, and administrators.

Swing UI SQLite JDK 25 Featured project
View on GitHub Read the README

Project summary

The app is designed around a single hotel property, with separate flows for guests, clerks, and administrators. That keeps the UI focused while still covering the core operational loop: account access, booking, stay management, and staff tooling.

What I built

  • Role-based flows - guest, clerk, and administrator experiences that route users to the right screens and tools.
  • Reservation workflow - booking and stay management with the data stored in SQLite.
  • Billing and account surfaces - guest billing, account center access, and supporting views for hotel operations.
  • Desktop-first UI - Swing-based interface built for a local application rather than a browser.
  • Launch-time data handling - opens or creates the SQLite database locally, with a fallback copy available from bundled resources.

Architecture

The project is organized around service and DAO layers so the hotel logic stays separated from the UI. That makes the reservation, billing, and account flows easier to reason about and extend.

Why it stands out

Unlike a web demo, this is a full desktop client with persistence, role-aware behavior, and real launch-time setup concerns. It reads like a compact operations tool instead of a toy example.

Technical notes

The README describes JDK 25, Maven 3.x, SQLite storage, and optional configuration for external services. The application also supports packaged distribution as a runnable jar-with-dependencies.

Result: Six Stars Hotel became the main featured project on the site because it shows a complete desktop application with a stronger systems feel than a simple landing-page demo.
