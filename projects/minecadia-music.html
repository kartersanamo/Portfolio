Minecadia Music

A Discord music bot for Minecadia with a Lavalink-powered player, an in-channel /music panel, a full web dashboard, and Discord Activity support—so members can queue, control playback, and launch the dashboard without leaving the server.

Overview

Music was split out of Utilities into its own application so playback, Lavalink, OAuth, and the dashboard could evolve independently. The bot exposes a Components V2 /music panel in Discord, streams from YouTube and SoundCloud through Lavalink, and serves a production dashboard at music.kartersanamo.com—including launch as a Discord Activity from the panel or app launcher.

What I built

  • In-channel player — /music panel with queue and transport controls using Discord’s component system.
  • Lavalink backend — native or Docker Lavalink on port 2333 for reliable multi-source playback.
  • Web dashboard — OAuth-backed UI proxied through nginx to the bot’s HTTP server (default port 8790).
  • Discord Activity — portal URL mappings and Launch Dashboard flow so the same UI runs inside Discord on web and mobile.
  • Production ops — dedicated Discord app, guild slash sync, voice-channel status hints, and fleet deployment alongside other Minecadia bots.

Stack

Python bot (cogs/, services/, ui/views) plus a JavaScript dashboard, Lavalink config under lavalink/, and docker/ compose for optional containerized audio nodes.

Configuration

.env holds token, database, Lavalink, and OAuth vars; assets/config.json covers roles, music settings, and presence. See .env.example for the full production checklist.

Technical notes

Migrating music to a standalone bot required coordinating Developer Portal Activities setup (Web/iOS/Android platforms, URL mappings, OAuth redirects) and de-registering the old /music command from Utilities via /utilities-sync after cutover.

Takeaway: a music bot is really three products—Discord UX, audio infrastructure (Lavalink), and a web surface (dashboard + Activity)—and they only feel seamless when OAuth, voice permissions, and nginx routing are planned together.
